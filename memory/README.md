# Agent Memory Vault

This folder is for agentic development context.

agent-arche treats `memory/` as an Obsidian-style Markdown vault for durable project knowledge: decisions, implementation patterns, learnings, reviews, feature notes, and session summaries. Agents read it before work and update it before finishing so future Copilot, Claude, or Codex sessions can share the same project memory.

This is not application runtime data, user content, or generated build output. Keep it in version control when your team wants AI assistants to share project context.

Start with `_MOC.md` as the map of contents.
